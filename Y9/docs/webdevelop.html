Section 1 Design Specification


The product will complete the following strands of the success critera:


Solves the Problem

The solution should be used by the S.A.S and serve a consistent and meaningful purpose and be useful to those who use the S.A.S. as wel as those who manage it. Since the product was requested, it would be helpful to make the product good enough that it will actually be used by the students.

User Experience

On top of solving the problem, the solution should also look good and feel modern. Using the latest web technologies and semantic code and lots of error checking, the product shouldn't be breakable and have many security measures such as firestore security rules to ensure that it will consistently work properly for everyone.

Accessible

Recently, I learned about keeping websites accessible. This means both making it mobile optimized and keeping the contrast levels good and following tested design principles such as material design will be vital in creating a usable product for everyone.
