Success Critera


Note: This success critera is from developing ideas and does not count as creating the solution.


I will know my solution is a success when...


  • Visuals
  • Graphics
  • Use



- Everything on the page is marginalized correctly.

- Everything follows similar design standards with the same color scheme consistent.

- There is a balance of typography, elements and white space.

- Everything on the page has a purpose and nothing is distracting.

- The entire visual aspect comes together as one to create a unifying and eye-opening experience.

- The visuals do not limit the impact the issue has on people




- The graphs are helpful and informing.

- The graphs are interactive and you are able to customize what is on the graph while outputting accurate data.

- The graphs are connected seamlessly.

- There is a balance of images, text and graphs.

- The images do not clash with the color scheme and theme.

- There are graphics such as arrows and guides only when neccessary.

- The page is not crowded with graphics.




!-! The entire solution is easy to use and can be used by ANYONE.

- The entire solution can be customized to the user's liking.

- It is easy to input your own countries / year / gho into the graphs.

- 5/5 people in the class can use the website with no problems.

- The user can get a good web experience from it while having being impacted by the issue.

- The use of the website does not detract from the impact the issue has on people.




Did I follow it?


Yes, I believe that my solution is successful based on my success critera!
The first section which is visuals. Everything on the page is marginalized correctly because I used the same "template" with the same size and style which turns into a clean look. Everything follows material design, and yes there is definitely a balance between typeography, content and white space. I tried my best to elimate everything that is unneeded on the page so I think I did well there. Everything is unyfying and nothing is distracting. And yes there is no problem with the visuals that I can see that will negatively affect the issue's impact on the users.

The next section is the graphics and similarly to the visuals section, I think that my solution does follow the success critera. This is because I do think that the graphs are informing because it is the best way to transmit specific results like mine. I did make it so that you can customize the graphs by adding an option to type in whichever country you would want to see results for as well as the year and GHO. There is a balance of images and they did follow the overall theme being material and using lighter colours instead of solid dark colours which would not go well. There are guides for how to use the various tools which will allow users to view data even if they do not know how to use it. Finally, there is only one spot on each page to show the graphics so it is definitely not crowded.

The final section in the success critera is use which basically says that the solution should be easy to use and be able to be used by anyone. It is very easy to input your own queries to get custom data and everyone who I have shown it to has been able to use without any major difficulties. In my humble opinion, it is definitely easy to use and I see no difficulties with how it's used based on performance from others. The difficulty of use does not affect how to issue impacts the user and will likely spread awareness regardless. Finally, I believe the website will not ddetract from the impact that the issue has on people and will instead enhance the issue by giving relavant and correct data.



In conclusion, my solution was a success based on my success critera which I made in developing ideas, due to the reasons above which explain why they were a success comparing it to the original success critera.

Create your own!

Although this was taught in class, I built it by myself to be able to be used in autocomplete with custom CSS. I used a for loop to search each line with a specific GHO and a specific Year and if the GHO equals a certain GUI and the year equals a certain year, then log the country. Naturally, there will only be one entry with a certain gui and a certain year for each country so I didn't need to worry about duplicated. ChartJS

It was challenging to get chartJS to work including compare functions to allow multiple countries and all of these values to be converted into integers and imported into a line chart or bar graph. Because I made this by myself first, before I had learned much, it was a lot harder that I thought. I went through various pages and I had to combine results to create my recursive search. Because I made it on my own it is different than the one which was shown in class.
