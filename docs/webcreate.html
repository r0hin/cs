Skills Demonstrated Section 2




HTML / CSS / Javascript

If Statements

"If" statements are integral to the code used to create this webapp. Everything from seeing if you are a teacher to knowing if you're signed in. When using inconjunction with firebase, you can do neat stuff with firestore and authentication. This is a very basic javascript module so I won't go into the specifics.

For Loops

For loops are the most basic loop. I used this a lot in my code mainly for things such as looping through the documents or workouts in this case and for each one, building a new element or set of elements in order to display them to the user. This is used both in the teacher and student view with different content to allow for the difference in functionality.

Functions & Events

Functions make up most of the app's javascript and it would be VERY difficult to create a semi-decent application without using functions. They not only help with organizing code, they also massively help with timing things and running code on an event. Things like onclick, onchange are all useful events or even on page load with JQuery. I use lots of these and virtually all my javsacript are built from functions. My code is on gitlab which you can see for yourself.

Syntax

Syntax and indentation is very lenient with Javascript especially with indentation. It is not like python where each indent matters and could make or break your code. It is still good practice to respect indentation in javascript which is what I do and is available on my gitlab page for the project where you can see the indentation working. In fact, javascript is so lenient that you could have everything on one line separated by semicolons!

Canvas & Grid

Canvas and CSS Grid are very useful content managers which allow you to make custom content or custom positions. The product has or will have both of these. The canvas is used for things such as graphs and infographics with stylesheets and libraries such as chartjs and codepen pens from the community. Grid was used for many of the content builders such as the list of workouts for the admin and student view. I will provide some examples as screenshots so you can see how they work.

User Experience

The user experience and UI is largely determined by how elements function and how they're positioned on the page. To keep everything looking nice, things are sometimes buried in modals or submenus, but to keep user experience high, everything is easily accessible and easily found so the user will not spend much time trying to find what they are looking for. I have also followed google's material design which I will provide screenshots of which has many designers on perfecting the design system.



Firebase

Authentication

The first firebase solution I used was the authentication feature which allows users to sign in with google. This will allow the website to reach it's entire target audience which is people who have a ucc.on.ca gmail. All other emails are blocked with is also done using firebase authentication and gtrabbing a users' profile. Authentication is also used to convert between teacher view and student view and determining if the user even has access to the teacher view. It check the user profile and gets user.email and if it matches a predefined set of emails, then you will be allowed in - all done using firebase authentication and if statements.

Firestore

Firestore is the database portion of firebase which I use. It is used to save user data and workout information to the cloud to be accessed on anyones computer and be modified at anytime from any device. This will allow teachers to actually save the workouts and upload it to the cloud. This is the main portion of the application since it allows the teacher to share the workout information with all the students at once. I used firestore to create a document for each workout and update it as such. Then get a snapshot of the collection and for loop through each document to build the user interface in order to show up on the admin or student panel for their respective purposes.

Web Development Section 5




The Nature of Web Dev

HTML / CSS / JS are the three languages of building websites. HTML decides what's on the pagee, CSS changes how it looks based on conditions such as default, hover, active and more. Javascript can dynamically run code and change the website. These three come together to create many webpages you are familiar with. Javascript is by far the most important as you can create HTML and CSS using javascript! If the website is the island, then your browser is the boat you use to get there. It doesn't matter what boat you use but some older boats (older browsers) will have some issues getting there correctly. HTML tags define what goes on the page. For example p tag is a paragraph tag and everything in between the opening bracket and the closing will be a paragraph element. This is the basics of web dev and web standards.



Programming Topics

Asynchronous Code

When getting data from firestore, it creates a promise. These promises will return data whenever that data is recieved from the database. Unfortunately, this is not instant and can take some time even if it's only a few miliseconds. If you have one promise and immediately after, you get that data and use it for something, it won't work because while the promise is still being recieved, javascript when onto the next piece of the code. So, to make things as asynchronous as possible, I use one of two options: .then() which allows you to right code after the promise is recieved, or using arrays. Using .then is much easier but doesn't work for many types of data especially when you have multiple documents. Using arrays essentially means, each time around you add it to the array, then you use a for loop for that array which essentially ensures that you have the data before you do anything with it. These are the methods of async promises and I have used this concept in many of my firebase apps including this one.

Error-handling

A famous tweet broke the twitter apps of thousands of users and created a self-liking tweet. How? Because the code author of a popular twitter client forgot to implement error checking. The tweet contained a script tag which would like it's own tweet. This is a lesson to always remember error checking or people can really abuse it. Try statements are one way to error check - you can filter our errors and run code despite possible errors. This is great if you are expecting an error and it's good to try the code even if it doesn't work. But, it is not good to wrap your entire code in a try statement because a lot of the code won't even work once the first error occurs. Secondly, you can use if statements. This is the method that I went with. To make sure the inputs didn't get too long, numbers only and restrictions on what can written to the database allows this application to run safely without the risk of people abusing it.

Document Object Model

The document object model or DOM is what allows anyone to use javascript to change the content on HTML. Things like grabbing things by ids and editing content from there is using the DOM, changing elements, attributed, styles and more is all DOM. Creating events and more is also all DOM. What I use the DOM in javascript for is very crutial for this app. It displays all the workouts in a grid. How would I do that? Well, I would first get a list of all the workouts with firebase firestore. I would call a snapshot for the collection and for each document, use the DOM to create an element and display it on the page with it's own content depending on the document content. Overall, javascript would not be remotely close to what it is today without the DOM and it is a W3C standard (World Wide Web Consortium).
