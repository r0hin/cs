Client & Problem

My client is William. He is an ordinary programmer who may or may not do shady things. This may or may not be why he is very concered with the privacy limitations when it comes to using traditional social media. This is an issue many internet users struggle with and there is no obvious choice for consumers. As modern social media apps are really bad at protecting their users' data, there could be lots of potential for a app which doesn't follow these bad trends.

As you can see from above, my solution is to create an app which doesn't track users or gather a lot of data about them. It should be really easy to use with a clean interface.

What is the project?

The project is to create a simple social media application that does not track you. It will provide a way to connect with people from around the world without giving away your privacy to big corporations. I can do this by making it not show their name if they don't want to.

Why undertake the project?

I believe that this project will be a good experience to enhance my abilities with Python. It is also a good way to learn more about Firebase and Pyrebase in particular and overall get the experience for Python.


Research Plan



Primary Skills



Python

Python is obviously the most important part of creating my solution because it is the language that I will be coding the entire solution in. This is why it is more than necessary to learn about the basic components of python that will be crucial to creating this project successfully.
Learning Source

Loops

Loops are crucial because they allow code to be run multiple times. This is useful for many things which I will be using for my solution such as using loops to create the posts and much more.
Learning Source

Functions

Functions allow you run code multiple times with different variables and predefefined actions. Most of my code will be in functions, that way, things can be run after each other, etc.
Learning Source

Imports

Imports are also important because it would be way too much to have to rewrite firebase for python, or not use the many libraries which I plan to use, so importing different codebases for example Pyrebase which I will talk about more later is exceptionally useful and required for my project to be remotely successful.
Learning Source

Firebase

Firebase is the service I will be using to allow multiple users to communicate with each other which will allow my solution to work on multiple computers and allow it to be successful. Firebase is a cloud functions service which makes creating databases easy and accessible. From past experience, I also have a good experience using it.
Learning Source

Authentication

I will be using Firebase authentication to log in users to prevent spam and allow users to skip the process of creating their profile, etc. I will be using sign in with email and password because it will be more accessible to those without a Google account (or other service account). It will also prove to be more data-secure because users know that Google shares a lot of data with developers, and that has shown to be important with my client.
Learning Source

Realtime Database

The realtime database is what I will be using to actually store data in the cloud. Unfortunately I cannot use Cloud Firestore because it is not supported by Pyrebase, and that is why I will need to learn about their only option which is by learning how to use the Realtime Database.
Learning Source

Third Party Libraries

Pyrebase

Pyrebase is what I will be using to bridge the gap of Firebase and Python. This will allow me to actually do everything I outlined before in a more secure and efficient way. It is a python wrapper which supports the Firebase API, I have run some tests to confirm that it works and it does. However, the syntax and general use of it differs from my comfort, therefore more learning is required to complete the solution for my client successfully. Learning Source

Tkinter

Tkinter is what I will be using to create the GUI and learning about this is very important because the alternative is that it will be command line based which is really not accessible and will probably be very non-user-friendly. Since we want this as accessible (and as polished) as possible, tkinter is the way I will be created my solution. Learning about the Tkinter widgets and their functions to create a good looking GUI will be worth the investement.
Learning Source

Design Brief



This solution is for William Banquer, a privacy concered individual who wants to protect his privacy while being online and being able to talk to people from around the world. He is 14 years old and in Grade 9 and the client for this project.
The solution is a high quality, lightweight, working and cloud focused product which allowed William Banquer to connect with people.
This product will be located on the computer and be accessible to all users, it will also be very lightweight because it is built with python and tkinter not required much else.
The client has specificed that this project should take no longer than a few months. I expect to have it completed by then, but after that, as long as the Firebase project is still running, It can run forever.
This product will allow privacy-concerned indivudual to connect with other people without risking losing all their data to big companies like Facebook and Google how have not had the best reputation for managing their users' personal data.
This product will be build with Firebase and Tinter using Python. Because it will be built with Python, it will be very lightweight (on mac computers) and generally be fairly cross compatitble as many devices natively support python.
